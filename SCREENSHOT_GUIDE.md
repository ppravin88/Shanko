# Screenshot Capture Guide for Shanko Card Game

This guide provides step-by-step instructions for capturing high-quality screenshots for marketing and documentation.

## Prerequisites

- Game running locally or deployed
- Modern web browser (Chrome, Firefox, or Edge recommended)
- Image editing software (optional): GIMP, Photoshop, or online tools

## Detailed Instructions

### 1. Game Setup Screen

**Purpose:** Show the initial player configuration interface

**Steps:**
1. Open the game
2. Stay on the setup screen (don't start a game yet)
3. Configure 4 players (2 human, 2 AI) for a balanced look
4. Ensure all UI elements are visible
5. Capture screenshot

**Key Elements to Show:**
- Player count selector
- Human vs AI toggles
- Player name inputs
- Start game button
- Deck count indicator

**Filename:** `screenshot-01-setup.png`

---

### 2. Game Board - Early Round

**Purpose:** Show basic gameplay in an early round

**Steps:**
1. Start a new game with 4 players
2. Play through Round 1 until you have a decent hand
3. Ensure you have some cards that could form combinations
4. Make sure the discard pile has a few cards
5. Capture during your turn (so controls are active)

**Key Elements to Show:**
- Player hand at bottom (7-11 cards)
- Draw pile (left side)
- Discard pile with visible top card (right side)
- Score board showing all players
- Round indicator (Round 1 or 2)
- Game controls (Draw, Discard, Meld buttons)
- Current player indicator

**Filename:** `screenshot-02-gameplay-early.png`

## Technical Specifications

### Resolution and Format

**Desktop Screenshots:**
- Resolution: 1920x1080 (Full HD)
- Format: PNG
- Color depth: 24-bit (RGB)
- DPI: 72 (web) or 300 (print)

**Mobile Screenshots:**
- Resolution: 1080x1920 (portrait) or 1920x1080 (landscape)
- Format: PNG
- Same color depth and DPI as desktop

### Browser Setup for Best Results

1. **Use Chrome or Firefox** for consistent rendering
2. **Zoom level:** 100% (Ctrl+0 to reset)
3. **Clear browser cache** before capturing
4. **Disable browser extensions** that might interfere
5. **Use incognito/private mode** for clean capture

### Capturing Methods

#### Method 1: Browser DevTools (Recommended)

1. Open DevTools (F12)
2. Click "Toggle device toolbar" (Ctrl+Shift+M)
3. Set custom dimensions (1920x1080)
4. Click the screenshot icon (â‹® menu â†’ Capture screenshot)
5. Saves full-page screenshot automatically

#### Method 2: Windows Snipping Tool

1. Press Win + Shift + S
2. Select area to capture
3. Screenshot copied to clipboard
4. Paste into image editor
5. Save as PNG

#### Method 3: Mac Screenshot

1. Press Cmd + Shift + 4
2. Drag to select area
3. Screenshot saved to desktop
4. Rename and move to screenshots folder

#### Method 4: Browser Extensions

- **Awesome Screenshot** (Chrome/Firefox)
- **Nimbus Screenshot** (Chrome/Firefox)
- **FireShot** (Chrome/Firefox)

## Usage

### Where to Use Screenshots

1. **Website:** Hero image, feature showcase, gallery
2. **Social Media:** Posts, ads, cover images
3. **Press Kit:** Media downloads
4. **Documentation:** User guides, tutorials
5. **App Stores:** If you create mobile apps later
6. **Blog Posts:** Feature announcements, updates
7. **Email Marketing:** Newsletters, announcements

### Image Optimization

Before uploading to web:

1. **Compress images:**
   - Use TinyPNG, ImageOptim, or Squoosh
   - Target: <200KB per image for web
   - Maintain visual quality

2. **Create multiple sizes:**
   - Thumbnail: 400x225
   - Medium: 1280x720
   - Full: 1920x1080

3. **Use appropriate formats:**
   - PNG: Screenshots with text/UI
   - JPG: Photographic content
   - WebP: Modern browsers (best compression)

## Troubleshooting

### Issue: Blurry Screenshots

**Solution:**
- Ensure browser zoom is 100%
- Use DevTools screenshot feature
- Capture at higher resolution and downscale

### Issue: Inconsistent Colors

**Solution:**
- Use same browser for all screenshots
- Disable color management in browser
- Use color picker to verify consistency

### Issue: UI Elements Cut Off

**Solution:**
- Increase viewport size
- Scroll to ensure all elements visible
- Use full-page screenshot option

### Issue: Large File Sizes

**Solution:**
- Use PNG compression tools
- Reduce color depth if possible
- Crop unnecessary areas

## Automation (Advanced)

For consistent screenshots, consider automation:

### Puppeteer Script Example

```javascript
const puppeteer = require('puppeteer');

async function captureScreenshots() {
  const browser = await puppeteer.launch();
  const page = await browser.newPage();
  
  await page.setViewport({ width: 1920, height: 1080 });
  await page.goto('http://localhost:5173');
  
  // Capture setup screen
  await page.screenshot({ 
    path: 'screenshot-01-setup.png',
    fullPage: false 
  });
  
  // Start game and capture gameplay
  await page.click('#start-game-button');
  await page.waitForTimeout(2000);
  await page.screenshot({ 
    path: 'screenshot-02-gameplay.png' 
  });
  
  await browser.close();
}

captureScreenshots();
```

- **Highlight features:** Each screenshot should showcase something specific
- **Keep updated:** Retake screenshots when UI changes
- **Get feedback:** Ask others if screenshots are clear and appealing
